# Contributing to Trunk Recorder AI Dashboard

Thank you for your interest in contributing to this project.

This repository aims to provide a lightweight real-time dashboard and transcription interface for Trunk Recorder systems. Contributions from the community are welcome and help improve the project for everyone.

---

## Ways to Contribute

There are many ways you can help improve the project:

• Reporting bugs
• Suggesting new features
• Improving documentation
• Submitting bug fixes
• Enhancing the dashboard interface
• Improving Docker deployment or configuration

If you're unsure where to start, reviewing open issues is a good first step.

---

## Reporting Issues

If you discover a bug or problem, please open a GitHub Issue.

When reporting an issue, try to include:

• A clear description of the problem
• Steps to reproduce the issue
• Expected behavior vs actual behavior
• Relevant logs or screenshots
• Your system environment (OS, Docker version, etc.)

This helps maintainers diagnose the issue quickly.

---

## Submitting Pull Requests

Pull requests are welcome.

Before submitting a PR, please ensure:

• Your changes are clearly documented
• The change addresses a specific issue or improvement
• The repository structure and formatting remain consistent
• Any new configuration options are documented in the README

Pull requests should focus on a **single logical change** whenever possible.

---

## Development Guidelines

When modifying code or configuration:

• Keep the project lightweight and easy to deploy
• Maintain compatibility with Trunk Recorder and Trunk Watcher workflows
• Avoid introducing unnecessary dependencies
• Ensure Docker configurations remain simple and portable

---

## Documentation Improvements

Documentation contributions are especially appreciated.

Examples include:

• clarifying setup steps
• improving troubleshooting instructions
• adding diagrams or screenshots
• improving explanations for new users

---

## Code of Conduct

Please be respectful and constructive when interacting with others in the project.

This project is intended to support the broader radio monitoring and homelab communities.

---

## Questions

If you have questions about contributing, feel free to open a discussion or issue in the repository.

Thank you for helping improve the project.

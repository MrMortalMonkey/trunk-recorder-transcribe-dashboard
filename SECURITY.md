# Security Policy

## Supported Versions

This project is actively maintained on the latest version available in the main branch.

Users are encouraged to keep their deployments up to date with the latest changes and security improvements.

---

## Reporting a Vulnerability

If you discover a security vulnerability related to this project, please report it responsibly.

Please include the following information when reporting:

• Description of the vulnerability
• Steps required to reproduce the issue
• Potential impact
• Any suggested mitigations

Security issues can be reported by opening a private GitHub security advisory or contacting the repository maintainer.

Please avoid publicly disclosing vulnerabilities until they have been reviewed and addressed.

---

## Scope

This project primarily consists of:

• a web dashboard interface
• Docker deployment configuration
• integration with Trunk Recorder and Trunk Watcher

Security considerations mainly involve:

• Docker container configuration
• API exposure
• external transcription services

Users should ensure that deployments are placed behind appropriate network protections when running in production environments.

---

## Responsible Disclosure

We appreciate responsible disclosure of security issues and will work to address confirmed vulnerabilities as quickly as possible.

Thank you for helping keep the project and its users safe.
